# 排查应用未安装问题

## 问题现象
- Android Studio 中能看到设备
- 运行 app 后，手机上没看到最新的应用

## 可能的原因和解决方法

### 1. 检查应用是否已安装（但没自动启动）

应用可能已经安装，但 Android Studio 没有自动启动它。

**解决方法：**
1. 在手机上打开应用列表/应用抽屉
2. 查找应用名称（通常是 "PhotoTimeGrouper" 或你在 `strings.xml` 中设置的名称）
3. 如果找到了，手动点击打开

### 2. 检查 Build 输出

**步骤：**
1. 在 Android Studio 底部查看 `Build` 面板
2. 查找是否有红色错误信息
3. 查找是否有 "INSTALL_FAILED" 相关的错误
4. 查看是否有 "Successfully installed" 或类似的成功消息

**常见的安装错误：**
- `INSTALL_FAILED_INSUFFICIENT_STORAGE` - 存储空间不足
- `INSTALL_FAILED_UPDATE_INCOMPATIBLE` - 版本不兼容（需要卸载旧版本）
- `INSTALL_FAILED_ALREADY_EXISTS` - 应用已存在但签名不同

### 3. 卸载旧版本应用

如果设备上已有旧版本的应用，可能会冲突。

**解决方法：**
1. 在手机上找到应用
2. 长按应用图标
3. 选择"卸载"或拖动到卸载区域
4. 在 Android Studio 中重新运行应用

**或者在 Android Studio 中卸载：**
1. 点击 `Run` → `Edit Configurations...`
2. 或者在 `Run` 面板的配置下拉菜单中选择应用配置
3. 确保选中了正确的设备

### 4. 清理并重新构建

**步骤：**
1. `Build → Clean Project`
2. 等待清理完成
3. `Build → Rebuild Project`
4. 等待构建完成（查看底部状态栏）
5. 再次运行应用

### 5. 检查设备连接状态

**步骤：**
1. 在 Android Studio 底部状态栏查看设备信息
2. 确保设备显示为 "online"（在线状态）
3. 如果显示 "offline" 或 "unauthorized"：
   - 拔掉 USB 线重新连接
   - 在手机上确认 USB 调试授权对话框

### 6. 手动安装 APK（备选方案）

如果自动安装一直失败，可以手动安装：

**步骤：**
1. 在 Android Studio 中构建 APK：`Build → Build Bundle(s) / APK(s) → Build APK(s)`
2. 等待构建完成
3. 构建完成后会弹出通知，点击 "locate"（定位）
4. 或者手动找到 APK：`app/build/outputs/apk/debug/app-debug.apk`
5. 将 APK 复制到手机
6. 在手机上打开 APK 文件安装

### 7. 检查应用名称

应用可能安装成功，但名称不同。

**检查方法：**
1. 查看 `app/src/main/res/values/strings.xml` 中的 `app_name`
2. 在手机上查找对应的应用名称

### 8. 查看 Logcat 输出

**步骤：**
1. 在 Android Studio 底部点击 `Logcat` 标签
2. 过滤选择你的设备
3. 运行应用
4. 查看是否有错误信息

## 快速检查清单

- [ ] 应用是否已安装在手机上（查看应用列表）
- [ ] Build 输出是否有错误信息
- [ ] 设备是否显示为 "online"
- [ ] 是否需要卸载旧版本
- [ ] 存储空间是否充足
- [ ] Logcat 是否有错误信息

## 下一步

根据检查结果：
1. 如果应用已安装：手动打开应用
2. 如果有安装错误：根据错误信息解决
3. 如果构建失败：清理并重新构建
4. 如果一直失败：尝试手动安装 APK
