# 隐私政策使用说明

## 📋 概述

本文档说明如何在应用中使用隐私政策，以及如何发布隐私政策网页。

---

## 一、文件说明

### 1.1 已创建的文件

1. **`docs/隐私政策.md`**
   - Markdown 格式的隐私政策文档
   - 用于版本控制和文档管理

2. **`docs/隐私政策.html`**
   - HTML 格式的隐私政策网页
   - 可以直接在浏览器中打开
   - 可以部署到网页服务器

## 二、在应用中添加隐私政策链接

### ✅ 已完成

隐私政策链接已添加到应用中：
- ✅ 在右上角菜单中添加了"隐私政策"选项
- ✅ 点击后会打开浏览器查看隐私政策网页
- ✅ 需要在 `MainActivity.kt` 中更新实际的隐私政策 URL

### 2.1 更新隐私政策 URL

在 `MainActivity.kt` 中找到 `openPrivacyPolicy()` 方法，将 URL 替换为您实际发布的隐私政策地址：

```kotlin
private fun openPrivacyPolicy() {
    // TODO: 将此处 URL 替换为您实际发布的隐私政策网页地址
    val privacyPolicyUrl = "https://your-domain.com/privacy-policy.html"
    // ... 其他代码
}
```

### 2.2 其他实现方式（可选）

如果需要其他方式添加隐私政策入口，可以参考以下方式：

### 2.2 在关于页面添加（如果存在）

```kotlin
// 在关于页面添加隐私政策链接
binding.privacyPolicyLink.setOnClickListener {
    val intent = Intent(Intent.ACTION_VIEW).apply {
        data = Uri.parse("https://your-domain.com/privacy-policy.html")
    }
    startActivity(intent)
}
```

### 2.3 在首次启动时显示（可选）

```kotlin
// 在首次启动时显示隐私政策同意对话框
private fun showPrivacyPolicyDialog() {
    AlertDialog.Builder(this)
        .setTitle("隐私政策")
        .setMessage("请阅读并同意我们的隐私政策")
        .setPositiveButton("同意") { _, _ ->
            // 保存用户同意状态
            savePrivacyPolicyAccepted()
        }
        .setNeutralButton("查看隐私政策") { _, _ ->
            openPrivacyPolicy()
        }
        .setNegativeButton("不同意") { _, _ ->
            finish()
        }
        .show()
}
```

## 三、发布隐私政策网页

### 3.1 发布选项

#### 选项 1：GitHub Pages（免费，推荐）

1. 在 GitHub 仓库中创建 `docs` 目录（如果不存在）
2. 将 `隐私政策.html` 复制到 `docs/` 目录
3. 在 GitHub 仓库设置中启用 GitHub Pages
4. 选择 `docs` 目录作为源
5. 访问地址：`https://your-username.github.io/PhotoTimeGrouper/privacy-policy.html`

#### 选项 2：个人网站/服务器

1. 将 `隐私政策.html` 上传到您的网站服务器
2. 确保可以通过 HTTPS 访问
3. 记录完整的 URL 地址

#### 选项 3：其他静态网站托管

- Netlify
- Vercel
- Firebase Hosting
- 其他静态网站托管服务

### 3.2 更新 HTML 文件中的联系信息

在发布前，请更新 `隐私政策.html` 中的以下内容：

```html
<!-- 在"联系我们"部分更新 -->
<li><strong>邮箱</strong>：[请填写您的联系邮箱]</li>
<li><strong>GitHub</strong>：[如果项目在 GitHub 上，请填写仓库地址]</li>
```

## 七、常见问题

### Q1: 必须要有隐私政策网页吗？
**A**: 是的，特别是申请了敏感权限（如存储权限）的应用，应用商店通常要求提供隐私政策链接。

### Q2: 隐私政策可以只提供 Markdown 文件吗？
**A**: 应用商店通常要求提供可公开访问的网页链接，所以需要 HTML 版本。

### Q3: 隐私政策需要多详细？
**A**: 至少需要说明：
- 收集了哪些信息
- 如何使用这些信息
- 是否分享给第三方
- 用户的权利

### Q4: 可以只提供英文版吗？
**A**: 如果应用主要面向中国用户，建议提供中文版本。如果面向国际用户，可以提供多语言版本。
