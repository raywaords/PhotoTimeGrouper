# 快速修复指南

## 当前情况

您从 `lib\plugins` 目录复制的文件不正确，而且 Gradle 缓存目录中可能没有完整的 wrapper 文件。

## 最简单的解决方案

### 方案 1：让 Gradle 自动修复（推荐）

直接运行 `gradlew`，Gradle 会自动下载并修复 wrapper：

```cmd
cd C:\Users\leiyu\Desktop\PhotoTimeGrouper
gradlew --version
```

如果网络正常，Gradle 会自动下载并修复 wrapper 文件。

### 方案 2：手动下载 Gradle 完整版

1. **下载 Gradle 8.5 完整版**：
   - 阿里云镜像：https://mirrors.aliyun.com/gradle/gradle-8.5-bin.zip
   - 或官方：https://services.gradle.org/distributions/gradle-8.5-bin.zip

2. **解压后找到文件**：
   ```
   gradle-8.5\gradle\wrapper\gradle-wrapper.jar
   ```

3. **复制到项目**：
   ```
   C:\Users\leiyu\Desktop\PhotoTimeGrouper\gradle\wrapper\gradle-wrapper.jar
   ```

### 方案 3：从其他项目复制

如果您有其他正常工作的 Android 项目：

1. 找到该项目的 `gradle\wrapper\gradle-wrapper.jar`
2. 复制到当前项目
3. 确保文件大小约 60-70 KB

## 验证

修复后运行：

```cmd
gradlew --version
```

应该显示 Gradle 版本，而不是错误。

## 文件大小参考

- ✅ 正常：60,000 - 70,000 字节（60-70 KB）
- ❌ 异常：小于 50,000 字节（文件不完整）
